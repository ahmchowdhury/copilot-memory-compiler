### Session (10:00) - CI/CD Pipeline Design

**Context:** repo: myproject, cwd: /home/user/myproject

**Key Exchanges:**
- **User:** What's a good GitHub Actions setup for a Python FastAPI project?
- **Assistant:** Use a multi-stage pipeline: lint (ruff), type check (mypy), test (pytest with coverage), build (Docker), deploy. Run lint and tests in parallel for speed.
- **User:** Should I use Docker layer caching?
- **Assistant:** Yes, use `docker/build-push-action` with GitHub Actions cache. Order your Dockerfile layers so requirements.txt is copied and installed before the source code — this way the dependency layer is cached unless requirements change.

**Decisions Made:**
- GitHub Actions with parallel lint + test jobs
- Docker layer caching with requirements-first pattern
- Deploy to Azure Container Apps

**Lessons Learned:**
- Pin your GitHub Actions versions with SHA hashes, not tags, for supply chain security
- Use `--no-cache-dir` in pip install inside Docker to reduce image size
