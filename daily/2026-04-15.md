### Session (09:30) - Setting Up FastAPI Auth

**Context:** repo: myproject, cwd: /home/user/myproject

**Key Exchanges:**
- **User:** How should I handle JWT auth in FastAPI?
- **Assistant:** Use python-jose for JWT encoding/decoding. Create a dependency that extracts and validates the token from the Authorization header. Store the secret in environment variables, never hardcode it.
- **User:** Should I use cookies or headers for the token?
- **Assistant:** For API-first apps, use the Authorization header with Bearer scheme. For web apps with server-rendered pages, HttpOnly cookies are more secure against XSS.

**Decisions Made:**
- Chose Authorization header + Bearer scheme for the API
- Using python-jose over PyJWT for better algorithm support

**Lessons Learned:**
- Always set token expiration — default to 30 minutes for access tokens
- Refresh tokens should be stored server-side, not in localStorage

### Session (14:15) - Database Migration Strategy

**Context:** repo: myproject, cwd: /home/user/myproject

**Key Exchanges:**
- **User:** What's the best way to handle database migrations with SQLAlchemy?
- **Assistant:** Use Alembic for migrations. Run `alembic init` to set up, then `alembic revision --autogenerate` to create migrations from model changes.
- **User:** How do I handle data migrations vs schema migrations?
- **Assistant:** Keep them separate. Schema migrations in Alembic, data migrations as standalone scripts. This avoids coupling and makes rollbacks cleaner.

**Decisions Made:**
- Using Alembic for schema migrations
- Separate data migration scripts in a `scripts/migrations/` directory

**Lessons Learned:**
- Always review autogenerated migrations before applying — they can miss renamed columns
- Test migrations against a copy of production data before deploying
